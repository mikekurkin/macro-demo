# Macro Demo

Тестовое задание на позицию фуллстек-разработчика.


## Запуск приложения

Фронтенд- и бэкенд-части завернуты в докер-контейнеры.
Для запуска в дев-режиме есть docker-compose.

* `docker-compose up --build` — собрать и запустить контейнеры.

Фронтенд с мок-данными будет доступен по ссылке http://localhost:3000

## В проекте используются:

### Бэкенд:
- **`Django 5`**
- **`Graphene`**
- **`SQLite`**

### Фронтенд:
- **`React Router`**
- **`Tanstack Query`**
- **`GraphQL Codegen`**
- **`shadcn/ui`**
- **`react-hook-form`**
- **`zod`**

## Формулировка задания

Итак, задание подразумевает создание fullstack-приложения с обязательным использованием Django, ReactJS и TypeScript, любые другие технологии тоже можно использовать.

Backend приложения должен сохранять и передавать информацию о рабочих местах. Рабочие места описываются следующим образом:

- Наименование (текстовое значение, обязательное, уникальное)
- Описание (текстовое значение)
- IP-адрес (текстовое значение)

Frontend приложения - для внесения и отображения данных о рабочих местах.

Необходимо разместить на форме поля для ввода информации, а также она должна содержать переключатель описания типа рабочего места:
- Место с компьютером
- Место без компьютера

Если выбран тип «Место без компьютера», то форма должна работать с двумя полями: наименование и описание. Если выбран тип «Место с компьютером», то отображается дополнительное поле «IP-адрес».

Задание сформулировано достаточно открыто, соискатель может проинтерпретировать все неуточнённые моменты так как ему будет удобно. Но если захочется получить какое-то уточнение от нас, то мы с радостью ответим. Пишите нам при возникновении каких-либо вопросов!
